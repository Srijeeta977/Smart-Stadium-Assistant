# 🏟️ Smart Stadium Assistant  
### 🚀 Submission for Virtual: PromptWars

## 📌 Problem Statement

Large-scale sporting venues often face challenges like crowd congestion, long waiting times, and lack of real-time guidance for attendees. This leads to poor user experience and inefficiency in movement and coordination.

---

## 💡 Solution

Smart Stadium Assistant is a web-based system designed to enhance the physical event experience by providing real-time guidance, crowd insights, and intelligent recommendations to users inside a stadium.

---

## 🚀 Features

* 📍 **Crowd Monitoring** – Simulates real-time crowd density at different gates
* ⏱️ **Queue Estimation** – Predicts waiting times at food stalls and facilities
* 🤖 **Smart Assistant** – Provides context-based recommendations (entry, exit, food, etc.)
* 🎯 **Decision Logic** – Suggests optimal choices based on current conditions

---

## 🧠 Approach & Logic

The system uses simulated real-time data to represent crowd density across different areas of a stadium. Based on this data:

* The assistant identifies the least crowded entry points
* Estimates waiting times dynamically
* Provides optimized navigation suggestions

The logic is designed to mimic real-world intelligent decision-making systems used in smart venues.

---

## 🔗 Google Services Integration

This project is designed to integrate with **Google Gemini API** for advanced AI-powered responses.

Due to the frontend-only prototype constraints, a logic-based assistant is currently implemented. In a production environment, the system can securely connect to Gemini via a backend service.

## ⚙️ How to Run

1. Download or clone the repository

3. Interact with the features:

   * Check crowd status
   * Estimate queue times
   * Ask the assistant for guidance
